Fixing Course Material
If you spot clear errors in the lecture slides you can suggest merge requests through GitLab.

Make sure that you do merge requests from a different branch and make sure that it does not contain any exercise solutions. If you do a merge request from master->master and continue committing exercise solutions those will be included in the merge request.

Short instructions (note you need to use haaranl1 main repo at first line): 

$ git remote add upstream git@version.aalto.fi:oseppala/wsd2017.git
$ git fetch upstream
$ git checkout -b corrections upstream/master

Now you are in a different branch corrections and there are only the material files in your working directory. You can now add corrections, add and commit them with a descriptive message. 

$ git commit -m "Fix ..."
$ git push origin corrections

In future you can change branches between corrections and master using the git checkout. You need to checkout the master for working with your solutions. 

$ git status
$ git checkout master
$ git status

Now you should go to version.aalto.fi. In the oseppala main repo go to the tab Merge Requests. Enter and add a new merge request. On the left, select your own repo and branch corrections. On the right, select oseppala/wsd2019 and master.
